# DigitalBooking

# ![logoDb](uploads/e2b055c96c589f30e8a610a95341cd92/logoDb.png) **Digital Booking Equipo 10**

* **Resumen del proyecto**

  El proyecto consistía en desarrollar una pagina web con las herramientas aprendidas hasta el momento. Aplicando nuestras habilidades en cada sector que correspondía. Para ello el grupo asignado, se dividió en cada área para poder llevarlo a cabo.

  El sitio web diseñado **ofrece servicios de reservas online.**<span dir=""> </span>**Actuamos como intermediarios entre los clientes que quieren reservar un alojamiento y tu hotel, propiedad o alquiler temporal/vacacional**<span dir="">.</span> **Es el programa que posibilita que los huéspedes hagan sus reservas por medio de la página de Internet de cualquier hotel**<span dir="">, lo cual es especialmente importante porque la mayoría de los viajeros prefieren hacer este proceso en la web.</span>

## Metodología de trabajo

En un principio, analizamos el proyecto, de que se trataba, visualizamos las tareas y en base a eso observamos las distintas áreas en el que cada integrante del grupo podía desarrollarse. Nos organizamos de acuerdo a que sección se sentía cada uno más cómodo y en que parte podía aportar su conocimiento.

Luego, las áreas donde habían mas de una persona involucrada, se reunían diariamente y organizaban sus tareas con sus avisos de cuando se hacía push en la rama donde se estaba trabajando.

Además de esto, el grupo en su totalidad se reunía 1 vez por día para ver en que situación se encontraban y que conflictos había tenido si fuese el caso. Al principio, en el primer sprint, cada uno se desenvolvió en su área perfectamente y sin conflictos, luego en los sprints restantes, si hubo que intervenir en otras áreas para dar su aporte y lograr llegar con los issues correspondientes a cada sprint.

Luego de cada sprint, se realizaba un retrospectiva de como había funcionado todo, en la mayoría de los sprints el equipo manifestó aspectos muy positivos en cuanto al funcionamiento del equipo, como también el resultado obtenido. Solo hubieron uno unos pocos inconvenientes que también se hablaron de ello y que el equipo resolvió de manera óptima.

Por ultimo, utilizamos este flujo de trabajo de ramas en función de Gitlab para llevar a cabo el proyecto de manera organizada, donde las ramas de back y front son las que quedaron automatizadas con los pipelines.

![image](uploads/ed90fd90d042cabe08d8090974ad5bc5/image.png)

## Bitácora del proyecto

1. Primer Sprint: Supimos llevar este sprint a cabo, llegando a todas las issues que se presentaron. Además, el equipo trabajo muy bien en conjunto y creo que se organizó de la mejor manera, utilizando reuniones diarias y métodos de trabajo que llevaron a generar un buen clima de trabajo.
2. Segundo Sprint: Trabajamos de manera muy similar al primer sprint, también evaluamos las cosas que habían por mejorar como también las que se destacaron de manera positiva. El equipo en su totalidad noto un compromiso muy bueno de parte de todos los participantes y que todos querían llegar a presentar un buen proyecto.
3. Tercer Sprint: Hubieron las primeras complicaciones en el equipo, ajustados con el tiempo y no poder presentarlo como se quería, mas allá de eso, respondió de la mejor manera posible y se solucionaron con mas reuniones y llegando a acuerdos.
4. Cuarto Sprint: Logramos con el objetivo, el equipo lo supo llevar a cabo y pudo sobrepasar ante cualquier imprevisto, se acomodó lo que había que corregir de los anteriores sprints, y todos los integrantes quedaron conforme con el resultado obtenido como también del trabajo de cada uno de los integrantes del grupo.

Fortalezas del equipo

En cuanto a organización y predisposición a la hora de trabajar creo que es una de las cualidades que destaca a este grupo. Rápidamente se organizo en cuanto a que áreas debía trabajar cada uno y en cuanto a las tareas por realizar. Otra de las cualidades es la responsabilidad, cada uno se preocupó por llegar a terminar las tareas cuanto antes por si otra área de trabajo necesitaba esa parte para continuar y llegar a terminar con todas las issues para cada sprint.

Debilidades

En cuanto a esta sección, hubo un problema de comunicación, por temas extras, ya sea familiar, laboral, etc. No se llegó a entender bien como era la situación y no se logró a llegar bien al 3er sprint. Mas allá de este problema, el equipo respondió y todos los integrantes se pusieron a trabajar en el sector comprometido, y logró en menor medida poder presentarlo. Luego se siguió trabajando en ello y se logró el objetivo.

## **<span dir="">Tecnologías utilizadas</span>**

* <span dir="">Servicios / herramientas que se emplearon a la hora de realizar el desarrollo</span> del proyecto.
  * <span dir="">Gestión del proyecto</span>: Issues y boards de Gitlab.
  * <span dir="">Desarrollo del proyecto:</span>
    * <span dir="">IDEs</span>: Visual Studio Code, IntelliJ IDEA, MySQL Workbench, Selenium IDE.
    * <span dir="">Sistemas de control de versiones</span>: Git y Gitlab.
* <span dir="">Qué tecnologías utilizaron para el desarrollo del sitio</span>.
  * <span dir="">Front End</span>: Vite + React.
  * <span dir="">Back </span>End: Spring Boot, Lombok, OpenApi 3, JWT, Security, SMTP Gmail.
  * <span dir="">Base de datos</span>: MySQL y RDS.
  * <span dir="">Infraestructura</span>: VPC, S3, RDS, Docker, CI/CD Gitlab.
  * <span dir="">Testing</span>: Backend: Junit5 y Mockito; Frontend : React testing library y Jest; API Testing:Postman y Swagger.
